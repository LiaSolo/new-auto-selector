# React + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Babel](https://babeljs.io/) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh

## Expanding the ESLint configuration

If you are developing a production application, we recommend using TypeScript with type-aware lint rules enabled. Пользовательский сценарий
1. При загрузке страниц *-settings в переменную useState считывается информация из соответствующих файликов *-final.json
2. Далее можно тыкать (менять) факультеты-участники, выбор попадает в ту же переменную useState 

!!! Нововведение: сразу выбор для обоих полуфиналов: выбираем для первого, остальные автоматически попадают во второй факультет, но можно и убрать какие-нибудь вовсе или переместить

файлики: semi1.json, semi2.json, final.json
при загрузке из файлов: если есть в каком-нибудь semi, то название активно и свитч в соответствующую сторону, если нет ни в каком, то название зачеркнуто, свитч дизейблед

есть две колонки "кто участвует" и "кто проходит дальше"
1. в первой находится список всех факультетов из конфига с чекбоксами
снизу  кнопка отправить учстников
useState [participiants, setPartisipiants]
2. после выбора участников и нажатия на кнопку происходит запись в файлик semi.json
и выбранные отображаются в правой колонке с новыми чекбоксами
снизу кнопка решить судьбу унивидения
useState [winners, setWinners]
3. под списком кто проходит дальше есть еще список победителей
после выбора победителей и нажатию на кнопку под списком кто проходит дальше выбранные участники попадают в третий список и записываются в файлик
он содержит названия факультетов и рядом с каждым кнопку "выпустить"
под всем списком кнопка продвинуть сюжет
useState [waiting, setWaiting], init -- winners
4. По нажатию на кнопку продвинуть сюжет на экран зрителя выводится первый в списке факультет, он удаляется из useState waiting, и засписывается в файлик как shown
в списке победители название зачеркивается, кнопка становится дизейблед
5. По нажатию на кнопку выпустить вне зависимости от очереди на экран выходит тот факультет, который сейчас выпустили, поведение аналогично п.4
6. при перезагрузке зрительской странички берется инфа из файлика semi.json, поэтому никто из выпущенных факультетов не потеряется
7. при перезагрузке странички настроек  берется инфа из файлика semi.json и распределяется по трем спискам с соответствующими состояниями
7. также есть кнопки 
    - "очистить всё" -- очищаются все три списка в semi.json
    - "очистить экран" -- очищается только shown в semi.json

пример semi.json
{
    partisipiants: [1, 2, 3, 4, 5, 6],
    winners: [1, 2, 3, 4],
    shown: [1, 3]
}

!! идея попроще
все чекбоксы появляются сразу из useState, а для отправки на сервер одна кнопочка "отправить всё, что знаю"
при загрузке странички также берется инфа из джсона

есть сущность "факультет-сеттингс", которая состоит из двух чекбоксов (участвует, проходит, надо их наверное цветом различать) и названия факультета. В колонке "кто участвует" отображается только первый чекбокс. При активации чекбокса этот факультет сразу появляется в колонке "кто победил", но со вторым чекбоксом (изначально неактивным). то есть теперь факультет присутствует в обеих колонках. если чекбокс в первой снять, то факультет автоматически очистит второй чекбокс и пропадет из второй колонки

3. После завершения выбора нажать на кнопку "Отправить участников", переменная useState кладется в соответствующий json-файлик.
